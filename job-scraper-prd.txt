1. Executive Summary
JobFlow AI is a locally-hosted, free-to-run automation system that handles the end-to-end job application pipeline for a software engineer. It scrapes job listings from LinkedIn, Indeed, and Naukri in real time, uses an AI language model to tailor a resume to each specific job description, logs every application to a live Google Sheets tracker, and monitors the applicant's Gmail inbox to automatically update application statuses when replies arrive.

The system is designed to run entirely on a developer's local machine at zero recurring cost, using only free-tier tools and open-source software. When the local setup is validated, it can be migrated to a free Oracle Cloud VM to run 24/7 unattended.

2. Problem Statement
Applying for software engineering roles is a high-repetition, low-automation process that consumes dozens of hours per week. The core pain points are:

Job discovery is fragmented — listings are spread across LinkedIn, Naukri, Indeed, and company career pages with no unified view.
Every application ideally requires a tailored resume and cover letter, but doing this manually is extremely time-consuming.
Tracking application status across multiple platforms is error-prone when done in spreadsheets manually.
Missed emails from recruiters mean missed opportunities — there is no automatic status sync.

JobFlow AI eliminates all of these manual steps, letting the job seeker focus only on preparing for interviews.

3. Goals and Non-Goals
3.1 Goals
Scrape job listings from LinkedIn, Indeed, and Naukri automatically on demand.
Use a free AI model (via OpenRouter) to tailor the user's base resume to each job description.
Save each tailored resume to Google Drive with a structured naming convention.
Log all discovered jobs to a Google Sheets tracker with company, role, status, URL, and date.
Monitor Gmail for recruiter replies and automatically update the tracker status.
Run entirely on local hardware at zero ongoing cost.
Provide a clean n8n workflow UI as the operator dashboard.

3.2 Non-Goals
Fully automated form submission — the user reviews and submits each application manually.
Mobile-native application — the interface is a browser-based n8n dashboard.
Paid API or SaaS dependency — every component must have a free or self-hosted option.
Multi-user support — this is a single-user personal automation tool.
Interview scheduling automation — out of scope for v1.

5. System Architecture
The system consists of three loosely-coupled components that communicate over HTTP on localhost:

Job Scraper Service — a Python/Flask server that runs JobSpy and exposes a /scrape endpoint.
n8n Workflow Engine — the orchestration layer that calls the scraper, sends data to OpenRouter, and writes to Google Sheets/Drive.
Gmail Watcher Workflow — a scheduled n8n workflow that reads the inbox and syncs statuses back to the sheet.

Data flow end to end:

User triggers Workflow 1 in n8n↓n8n POST → Flask /scrape → JobSpy scrapes LinkedIn, Indeed, Naukri↓n8n splits job list → OpenRouter (Llama 3.3 70B free) tailors resume per JD↓Tailored resume saved to Google Drive · Row appended to Google Sheets↓User reviews Sheet → applies manually via job URL → updates Status to Applied↓Workflow 2 (hourly) reads Gmail → AI classifies email → Sheet status updated automatically

8. Data Model — Google Sheets Tracker
Each row in the Google Sheets tracker represents one job application. Columns:

Column | Type | Description
Company | String | Company name extracted from scraper
Role | String | Job title from the listing
Location | String | City / Remote as returned by JobSpy
Applied Date | Date | YYYY-MM-DD format, set at time of scrape
Status | Enum | Pending Review → Applied → Interview Scheduled → Offer → Rejected
Job URL | URL | Direct link to the job posting — used for deduplication
Resume Version | String | Google Drive file name of the tailored resume for this role
Notes | String | Auto-set to 'Auto-scraped'; user can overwrite
Salary Min | Number | Populated where available from the listing
Salary Max | Number | Populated where available from the listing
Source | String | LinkedIn / Indeed / Naukri


9. API Contracts
9.1 Flask Scraper — POST /scrape
Request body (JSON):

Field | Type | Description
keywords | string | Search term e.g. "Full Stack Developer OR AI Engineer"
location | string | "Hyderabad, India" — passed directly to JobSpy
sources | string[] | ["linkedin", "indeed", "naukri"] — any subset
num_jobs | integer | Max results per source. Default: 20
hours_old | integer | Filter to jobs posted within N hours. Default: 168 (7 days)


Response (JSON): { jobs: Job[], count: number } — where each Job object contains source, title, company, location, url, description, job_type, date_posted, is_remote, salary_min, salary_max.

11. Constraints and Risks
11.1 Constraints
Zero paid dependencies — every component must be free or open-source.
Single-user only — no authentication layer, multi-tenancy, or shared data.
Manual application step — job boards actively block fully automated form submission.
OpenRouter free tier — 200 requests per day, 20 per minute. Limits large daily scrape runs.

11.2 Risks and Mitigations
Risk | Likelihood | Mitigation
Job board blocks scraper IP | High | Add request delays; rotate user-agent strings; keep num_jobs low per run
LinkedIn HTML structure changes | Medium | JobSpy is actively maintained — update library regularly via pip
OpenRouter free model deprecated | Low | OpenRouter has multiple free fallback models; swap model ID in n8n body
Google OAuth token expiry | Low | Re-authenticate via n8n credentials page; tokens auto-refresh when valid
Gmail classifier misidentifies email | Medium | Prompt includes explicit JSON-only output; add fallback status 'Review Needed'
n8n Docker container resets data | Low | Use named Docker volume (n8n_data) to persist workflow and credentials data


12. Success Metrics
Scraper returns at least 15 valid job listings per run across selected sources.
AI-tailored resume is generated within 30 seconds per job description.
Google Sheets tracker is updated within 60 seconds of workflow completion.

Zero recurring monthly cost — all APIs and compute remain within free tiers.
User applies to at least 10 curated roles per day with less than 30 minutes of active effort.
